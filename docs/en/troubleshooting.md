# Troubleshooting

## Check Logs First

MCP calls are printed in the ComfyUI console with the `[MCP]` prefix:

```text
[MCP] list_templates() → 3 templates
[MCP] run_template(name='txt2img', params={"positive_prompt": "a cat"}) → completed
[MCP] upload_image(source=E:/photos/input.png) → {"name": "mcp_4b2f...a91c.png", "subfolder": "", "type": "input"}
```

Proxy requests use the `[MCP Proxy]` prefix:

```text
[MCP Proxy] POST /app-mcp → http://127.0.0.1:8189/mcp
[MCP Proxy] upstream error: ...
```

## Template List Is Empty

Check:

1. The workflow was saved to the ComfyUI server through **Save**.
2. The workflow was not only exported locally through **Export**.
3. The workflow was created as a template in **Settings → MCP Server → Templates**.
4. The template is not disabled. Disabled templates do not appear in `list_templates()`.

## Workflow Not Found When Creating a Template

Plugin mode automatically reads ComfyUI's actual startup port and accesses its API through the local loopback address.

To override the auto-detected address, set this before startup:

```bash
COMFYUI_URL=http://<comfyui-host>:<port>
```

For standalone deployments, you can also set `comfyui.apiUrl` in `mcp.config.json`. See [Standalone and Remote Access](./standalone.md).

## Outputs Are Empty

Check that output nodes are marked in App Builder. Templates prefer `linearData.outputs` from the workflow.

If output settings changed, click **Refresh** on the same-name template.

## Random Seed Does Not Work

Name the input `seed` if it should be randomized automatically. Runtime fills it automatically, `get_template()` does not expose it, and AI clients do not need to pass it.

## Parameter Mapping Errors

If ComfyUI reports `value_not_in_list`, widget value mapping is likely stale or misaligned.

Try:

1. Restart ComfyUI so the plugin reloads node definitions.
2. Click **Refresh** on the template.
3. Ask the AI to call `get_template()` again.

## Remote Access Returns 421

Check:

1. ComfyUI was started with `--listen`.
2. The MCP client URL uses a reachable LAN IP or domain.
3. If using a reverse proxy, it forwards to the correct ComfyUI or standalone MCP port.

## Image Input Does Not Work

Identify the image source:

- New image from the user: call `upload_image()` and use the returned `name` as the template parameter.
- Image generated by a template: do not upload it manually; let the AI use template chaining.

## Images Cannot Be Sent Through AstrBot or Similar Platforms

First check the argument types used when the AI calls the platform's send tool.

Common issues:

- Passing an image URL into a local-file argument such as `path` or `file_path`.
- The platform send tool expects `url` or `image_url`, but the AI passes a local-path field.
- The platform only accepts local files; in that case, download the image first and pass the local path.

Ask the AI to inspect the platform send tool schema again and pass `url`, `path`, or `file` according to the actual parameter meaning.

## Template Chaining Fails

Check:

1. Ask the AI to call `get_template()` again for the target template.
2. Confirm the previous step produced the needed image, audio, or text.
3. Ask the AI to follow [Tool Reference: Bindings](./tools.md#bindings) and retry the call.

## Generation Times Out

A `run_template(wait=true)` timeout does not mean the task failed. Use the returned `run_id` to continue:

```text
get_template_result(name, run_id, wait=true)
```

The default wait time can be changed in **Settings → MCP Server → Execution → Run Template Timeout**.
